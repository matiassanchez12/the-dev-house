# Page Spec: Terms

## Objetivo

Presentar terminos de uso genericos, breves y claros para una app de colaboracion en desarrollos de software comunitarios.

## Ruta y pagina

- Ruta: `/terms`
- Archivo React: `resources/js/pages/public/terms.tsx`
- Titulo SEO: `Terminos de uso | The Dev House`

## Estructura de contenido

1. Introduccion
   - Alcance general de la plataforma.
2. Uso aceptable
   - Respeto entre usuarios, contenido permitido, prohibiciones basicas.
3. Cuentas y responsabilidad
   - Datos reales/minimos, seguridad de acceso, responsabilidad por actividad.
4. Propiedad de contenido
   - Cada usuario conserva autoria de su contenido.
5. Limitacion de responsabilidad
   - Servicio brindado de forma general, sin garantias absolutas.
6. Cambios en terminos
   - Posibilidad de actualizar terminos con aviso razonable.
7. Contacto legal
   - Canal de contacto para consultas.

## Requisitos de UI

- Formato legible tipo documento: titulo + secciones + subtitulos.
- Espaciado amplio y ancho de lectura controlado.
- Evitar muro de texto, usar parrafos cortos.

## Criterios de aceptacion

- Texto breve, entendible y no ambiguo.
- Cobertura de los puntos legales basicos.
- Coherencia visual con el resto de paginas publicas.
